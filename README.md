# Balubo - ライター特化型AI分析ポートフォリオサービス

![Balubo Logo](https://via.placeholder.com/200x80?text=Balubo)

Baluboは、**ライターのために設計されたAI分析型ポートフォリオサービスです。あなたの執筆実績をAIが多角的に分析し、「オリジナリティ」「クオリティ」「エンゲージメント」の3軸で“あなたらしさ”と“信頼”を可視化。SNSや大手サービスでは伝わりにくい、ライター個人の魅力・強みを発注主に直接届けます。

---

## 🎯 ミッション（大義）
> **AIで生成できる時代に、“クリエイター（人）”を応援する！**

AIが文章やコンテンツを量産できる時代だからこそ、「人」ならではの価値や個性、信頼を可視化し、クリエイターの活躍を後押しします。

## 👤 ターゲット
- **コアターゲット：フリーランス・副業のライター**
  - SNSや大手プラットフォーム（ランサーズ・Facebook・LinkedIn等）では埋もれがちな、個人の“信頼”や“こだわり”を見せたい方
  - 仕事のやりとりに不安や面倒さを感じている方
- **優先順・今後拡張**：
  - カメラマン・デザイナー・イラストレーター等にも展開予定
- **発注側ターゲット**：
  - 個人商店・スモールビジネス・編集プロダクション・Webメディア担当者
  - 「信頼できるライター」「得意分野で選べるライター」を探してている方

---

## 💡 インサイト
- SNS経由の依頼はギャラや信頼性が不透明、知人経由は断りづらいなど、ライター特有の悩みが多い
- 実績や得意分野、媒体ごとの執筆経験を“安心して”見せたい・伝えたい
- 「どんな仕事をどんな媒体で、どんな評価を受けてきたか」を一目で伝えたい

---

## 🎁 ライター向けベネフィット
- **AIがあなたの執筆実績・ポートフォリオを3軸で分析・可視化**
  - オリジナリティ：あなたらしい視点・切り口
  - クオリティ：専門性や文章力・編集力
  - エンゲージメント：読者やクライアントへの影響力
- **自己PR・3つの質問・ショートインタビュー**で“人”としての魅力も発信
- **信用スコア・進行管理・ダイレクトプライシング**など、受発注の不安や面倒を減らす仕組み
- **発注主からの直接依頼・マッチング**で、単価や条件も自分でコントロール

---

## 🌟 主な特徴（ライター特化型）
- **作品登録**: 記事URLや原稿ファイル、画像で簡単登録
- **AI分析**: Google Gemini APIによるライター特化の日本語分析
- **タグ・媒体・ジャンル管理**: 得意分野や媒体ごとの実績をタグで可視化
- **直感的なダッシュボード**: 3軸分析・タグクラウド・媒体別実績集計
- **自己PR・ショートインタビュー**: “人”としての魅力やこだわりを伝える
- **信用スコア・進行管理**: 受発注の信頼性・やりやすさをサポート
- **ダイレクトプライシング**: 忙しさや実績に応じて単価を自分で調整
- **プロフィール編集・SNS連携**: 基本情報・SNS・実績管理
- **キャリア管理**: 職歴・執筆媒体の追加・編集・削除
- **レスポンシブデザイン**: PC・スマホで快適に利用可能

## 🚀 技術スタック
- **フロントエンド**: React (^19.0.0), TypeScript (^5.0.0), Tailwind CSS (^3.4.17)
- **バックエンド/DB**: Supabase (PostgreSQL)
- **認証**: Supabase Auth (Google認証含む)
- **ストレージ**: Supabase Storage
- **AI/ML**: Google Gemini API (claude-3-7-sonnet-20250219)
- **ホスティング**: Netlify（デプロイURL: https://balubo.netlify.app/）
- **CI/CD**: GitHub Actions

## 🛠 開発環境セットアップ

### 前提条件

- Node.js (^20.0.0)
- npm (^10.0.0)
- Supabaseアカウント
- Google Gemini APIキー

### インストール手順

1. リポジトリをクローン
   ```bash
   git clone https://github.com/yourusername/balubo_project.git
   cd balubo_project
   ```

2. 依存関係のインストール
   ```bash
   npm install
   ```

3. 環境変数の設定
   `.env.example`を`.env`にコピーし、下記を設定してください。
   ```bash
   cp .env.example .env
   ```
   - SUPABASE_URL
   - SUPABASE_ANON_KEY
   - SUPABASE_SERVICE_ROLE_KEY（Netlify Functions用はこの名称で統一）
   - GEMINI_API_KEY

4. 開発サーバーの起動
   ```bash
   npm run dev
   ```

## ⚠️ 注意事項・FAQ

- 画像アップロード時はSupabase Storageに`works`バケットが必要です。存在しない場合は自動作成されますが、RLSルールや公開設定を確認してください
- RLS/権限エラー発生時はSupabaseのポリシー設定を見直してください
- AI分析はGoogle Gemini API（claude-3-7-sonnet-20250219）を使用し、必ず日本語で出力されます
- Netlify Functions/APIのCORSや環境変数名の不一致に注意
- 詳細な仕様・設計は`/docs`やコードコメントを参照してください
