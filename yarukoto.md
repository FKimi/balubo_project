# Balubo - AI分析型ポートフォリオサービス 要件定義

## 1. プロジェクト概要

### 1.1 サービス定義
- **サービス名**: Balubo
- **概要**: ライター/クリエイター向けAI解析型ポートフォリオ自動生成サービス
- **コアバリュー**: URLを入力するだけでAIが作品を分析し、専門性・作品特徴・興味関心の3軸で可視化
- **開発期間**: 〜2025年3月25日まで

### 1.2 ターゲットユーザー
- **メイン**: フリーランスのライター・編集者
- **サブ**: デザイナー、イラストレーター、カメラマン
- **閲覧者**: クリエイターに仕事を依頼したい企業・個人

### 1.3 解決する課題

#### クリエイター側
- **ポートフォリオ作成・更新の手間**
  - 専門知識や時間をかけず、手軽に作品をまとめたい
- **客観的な自己PRの難しさ**
  - 自分の強み・専門性を外部にアピールしづらい
- **作品の散在**
  - 複数のメディアやサイトに掲載した作品を一元管理したい

#### 発注者側
- **適切な人材発見の難しさ**
  - クリエイターの実績や専門性を正確に把握しづらい
- **実績の信頼性評価**
  - 自己申告だけでなく客観的データを求めたい
- **予算・案件のマッチング**
  - クリエイターのレベルや専門性に合う依頼が難しい

### 1.4 本質的な課題
- ポートフォリオの作成・更新・作り込みを簡易化する
- 自分の経験やスキルを正確かつ客観的に示す負荷を下げる
- 散らばった作品を一元管理する

## 2. 技術スタック
- **フロントエンド**: React + TypeScript, Tailwind CSS
- **バックエンド/DB**: Supabase (PostgreSQL)
- **認証**: Supabase Auth (Google認証含む)
- **ストレージ**: Supabase Storage
- **AI/ML**: Google Gemini API (1.5 Flash)
- **ホスティング**: Netlify
- **その他**: Supabase Edge Functions (メタデータ取得)

## 3. 主要機能要件

### 3.1 認証・ユーザー管理
- メール/パスワードでのサインアップ・ログイン
- Googleアカウントによるソーシャルログイン
- プロフィール編集機能
- SNS連携機能（Twitter/X, Instagram, Facebook）

### 3.2 作品登録・管理
- URL入力による作品情報の自動取得（OGP活用）
- 作品タイプ分け（記事/デザイン）
- デザイン作品のファイルアップロード
- 作品編集・削除機能
- 公開/非公開設定
- フィルタリング・検索機能

### 3.3 AI分析機能
- **作品ごとの自動分析**
  - 専門性（強み）分析
  - 文章スタイル・特徴分析
  - 興味・関心領域分析
- **ユーザー総合分析**
  - タグベースの集約・分析
  - 才能スコアの算出
- **分析結果のビジュアライズ**
  - レーダーチャート（専門性）
  - バーチャート（スタイル）
  - タグクラウド（興味関心）

### 3.4 タグベース分析（重点機能）
- AIによる自動タグ提案
- タグの出現頻度分析
- 関連タグのクラスタリング
- タグの時系列変化分析
- インタラクティブなタグマップ

### 3.5 ポートフォリオ表示
- ユーザーダッシュボード
- 作品一覧表示（グリッド/リスト）
- 活動指標の視覚化
- 作品詳細ページ
- 共有可能なURL生成

### 3.6 サブスクリプション管理
- **無料プラン**
  - 基本ポートフォリオ機能
  - 最大10作品までAI分析
- **有料プラン（月額500円）**
  - AI分析の無制限化
  - 詳細AI分析ページ（アナリティクスページ）の開放

## 4. データベース設計
- **主要テーブル**:
  - users（ユーザー情報）
  - works（作品情報）
  - tags（タグ情報）
  - work_tags（作品-タグ関連付け）
  - ai_analyses（作品分析結果）
  - user_analyses（ユーザー総合分析）
  - tag_analytics（タグ分析詳細）

## 6. 非機能要件
- **性能**: ページロード2秒以内、AI分析5秒以内
- **セキュリティ**: RLS、API鍵管理、XSS/CSRF対策
- **可用性**: 99.9%以上、ユーザーあたり最大100MB
- **UX**: レスポンシブデザイン、直感的UI

## 7. KPI目標
- 登録ユーザー数: 初年度1,000人
- 作品登録数: 初年度10,000作品
- 有料プラン転換率: 5%
- ユーザー継続率: 3ヶ月後60%
- ポートフォリオ共有数: ユーザーあたり平均3回
- タグ活用度: 作品あたり平均5タグ以上

## 9. 実装アプローチ
- 段階的な開発を重視（小さく始めて検証）
- URLからの情報抽出と保存を最初に実装
- AI分析機能は順次追加
- タグベース分析を重点機能として優先実装
