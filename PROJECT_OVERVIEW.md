# Balubo プロジェクト概要

## 1. プロジェクト定義

Baluboは、ライターやクリエイター向けのAI解析型ポートフォリオ自動生成サービスです。ユーザーが作品のURLを入力するだけで、AIが作品を分析し、専門性・作品特徴・興味関心の3軸で可視化するポートフォリオを自動生成します。

### 1.1 コアバリュー

- **手間の削減**: 専門知識や時間をかけずに、ポートフォリオの作成・更新が可能
- **客観的な分析**: AIによる客観的な強み・専門性の分析と可視化
- **一元管理**: 複数のメディアやサイトに散在する作品を一箇所で管理

### 1.2 ターゲットユーザー

- **メイン**: フリーランスのライター・編集者
- **サブ**: デザイナー、イラストレーター、カメラマン
- **閲覧者**: クリエイターに仕事を依頼したい企業・個人

## 2. 技術スタック

### 2.1 フロントエンド
- **React + TypeScript**: 型安全なコンポーネント開発
- **Tailwind CSS**: 効率的なUIデザイン実装
- **shadcn/ui**: 高品質なUIコンポーネント

### 2.2 バックエンド
- **Supabase (PostgreSQL)**: データベース、認証、ストレージ
- **Netlify Functions**: サーバーレスAPI実装

### 2.3 AI/ML
- **Google Gemini API**: 作品分析、タグ生成、専門性評価
- **OpenAI API**: テキスト分析、要約生成（バックアップ）

### 2.4 インフラ
- **Netlify**: ホスティング、CI/CD
- **GitHub**: バージョン管理

## 3. 主要機能

### 3.1 ユーザー管理
- **認証**: メール/パスワード、Google、GitHub連携
- **プロフィール管理**: 基本情報、職歴、スキル設定
- **ダッシュボード**: 作品統計、アクティビティ表示

### 3.2 作品管理
- **作品登録**: URL入力による自動取得
- **メタデータ編集**: タイトル、説明、タグの編集
- **作品分類**: カテゴリー、タグによる整理

### 3.3 AI分析
- **テキスト分析**: 専門性、文体、トピック抽出
- **タグ分析**: 頻出タグ、特徴的なタグの抽出
- **専門性評価**: 専門分野の特定と評価
- **ポートフォリオ最適化**: 強みを活かした作品選定提案

### 3.4 レビュー機能
- **ピアレビュー**: 他のクリエイターからのフィードバック収集
- **評価指標**: 専門性、創造性、技術力などの多角的評価
- **コメント管理**: レビューコメントの表示・返信機能
- **評価統計**: 受けたレビューの統計データ可視化

### 3.5 ポートフォリオ表示
- **カスタマイズ可能なレイアウト**: テーマ、色、構成の変更
- **作品ギャラリー**: グリッド/リスト表示
- **専門性グラフ**: レーダーチャートによる可視化
- **共有機能**: 公開URL、埋め込みコード生成

# Balubo - AI解析型ポートフォリオサービス 最終要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト定義・目的
Baluboは、ライターを中心としたクリエイター向けのAI解析型ポートフォリオサービスです。作品URLを入力するだけで、タイトルや説明文を自動取得し、AIがそのコンテンツを分析することで、以下の3つの軸で魅力を可視化したポートフォリオを作成します。

- 「専門性（経験・強み）」
- 「作品の特徴・魅力」
- 「興味・関心」

ユーザーは自分の作品をまとめて一元管理でき、外部への共有も容易に行えます。

### 1.2 背景
クリエイターにとって、ポートフォリオの作成・更新は時間がかかる作業です。また、自身の強みを客観的に伝えることが難しく、複数のプラットフォームに散在した作品を一元管理する手段も限られています。本サービスはこれらの課題を解決します。

### 1.3 プロジェクトスコープ
- **開発期間**: 〜2025年3月25日まで
- **優先実装機能**: MVP（最小実用製品）の開発
- **対象プラットフォーム**: Webブラウザ（レスポンシブ対応）

### 1.4 ターゲットユーザー
- **メイン**: フリーランスのライター・編集者
- **サブ**: デザイナー、イラストレーター、カメラマン
- **閲覧者**: クリエイターに仕事を依頼したい企業・個人

### 1.5 解決する課題

#### クリエイター側
- **ポートフォリオ作成・更新の手間**
  - URL入力だけで作品情報を自動取得できるため、手間が削減される
- **客観的な自己PRの難しさ**
  - AIによる分析で専門性や特徴を客観的に可視化
- **作品の散在**
  - 複数メディアの作品を一カ所にまとめられる

#### 発注者側
- **適切な人材発見の難しさ**
  - クリエイターの専門性を可視化することで適切なマッチングを実現
- **実績の信頼性評価**
  - AI分析による客観的な評価指標を提供
- **予算・案件のマッチング**
  - クリエイターのレベルや専門性を把握しやすい

### 3.4 Row Level Security (RLS) ポリシー

#### users テーブル
- **読み取り**: 認証済みユーザーが自身のデータのみ読み取り可能
- **更新**: 認証済みユーザーが自身のデータのみ更新可能

#### works テーブル
- **読み取り**: 認証済みユーザーが全ての作品を閲覧可能
- **作成**: 認証済みユーザーが自身の作品のみ作成可能
- **更新**: 認証済みユーザーが自身の作品のみ更新可能
- **削除**: 認証済みユーザーが自身の作品のみ削除可能

#### tags テーブル
- **読み取り**: 全てのユーザーが読み取り可能

#### work_tags テーブル
- **読み取り**: 認証済みユーザーが自身の作品に関連するタグを読み取り可能
- **作成/削除**: 認証済みユーザーが自身の作品のタグのみ操作可能

#### ai_analyses テーブル
- **読み取り/作成/更新**: 認証済みユーザーが自身の作品に関連する分析のみ操作可能

#### user_analyses テーブル
- **読み取り/作成/更新**: 認証済みユーザーが自身の分析のみ操作可能

## 8. 非機能要件

### 8.1 性能要件
- ページロード時間: 3秒以内
- API応答時間: 2秒以内
- AI分析処理時間: 5秒以内
- 最大同時ユーザー数: 100名

### 8.2 セキュリティ要件
- データベースへのアクセス制御（RLS）
- 認証済みユーザーのみが自身のデータを操作可能
- APIキーのセキュアな管理（環境変数）
- XSS/CSRF対策の実装
- HTTPS通信の強制

### 8.3 可用性・拡張性要件
- 稼働率: 99.9%以上
- ストレージ容量: ユーザーあたり最大100MB
- データバックアップ: 日次
- モジュラー設計によるコード拡張性確保

### 8.4 制約・前提条件
- Gemini APIの利用制限
  - 入力トークン上限: 1,048,576
  - 出力トークン上限: 8,192
  - リクエスト数制限: 考慮必要
- Supabase無料プランの制限
  - ストレージ容量: 1GB
  - データベース容量: 500MB
  - 月間アクティブユーザー: 50,000
- Netlify無料プランの制限
  - ビルド時間: 月300分
  - サイトあたりのバンド幅: 100GB/月
