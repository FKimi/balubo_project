# Balubo プロジェクト概要

## 1. プロジェクト定義

Baluboは、ライターやクリエイター向けのAI解析型ポートフォリオ自動生成サービスです。ユーザーが作品のURLを入力するだけで、AIが作品を分析し、専門性・作品特徴・興味関心の3軸で可視化するポートフォリオを自動生成します。

### 1.1 コアバリュー

- **手間の削減**: 専門知識や時間をかけずに、ポートフォリオの作成・更新が可能
- **客観的な分析**: AIによる客観的な強み・専門性の分析と可視化
- **一元管理**: 複数のメディアやサイトに散在する作品を一箇所で管理

### 1.2 ターゲットユーザー

- **メイン**: フリーランスのライター・編集者
- **サブ**: デザイナー、イラストレーター、カメラマン
- **閲覧者**: クリエイターに仕事を依頼したい企業・個人

## 2. 技術スタック

### 2.1 フロントエンド
- **フレームワーク**: React + TypeScript
- **スタイリング**: Tailwind CSS
- **データ可視化**: Recharts

### 2.2 バックエンド
- **データベース**: Supabase (PostgreSQL)
- **認証**: Supabase Auth (Google認証含む)
- **ストレージ**: Supabase Storage
- **サーバーレス関数**: Supabase Edge Functions

### 2.3 AI/ML
- **テキスト分析**: Google Gemini API (1.5 Flash)
- **APIキー管理**: 環境変数経由 (VITE_GEMINI_API_KEY)

### 2.4 デプロイ
- **ホスティング**: Netlify
- **デプロイURL**: https://stupendous-llama-975208.netlify.app/
- **CORS対応**: Netlify Functions

## 3. 主要機能

### 3.1 認証・ユーザー管理
- メール/パスワードでのサインアップ・ログイン
- Googleアカウントによるソーシャルログイン
- プロフィール編集機能
- SNS連携機能（Twitter/X, Instagram, Facebook）

### 3.2 作品登録・管理
- URL入力による作品情報の自動取得（OGP活用）
- 作品タイプ分け（記事/デザイン）
- デザイン作品のファイルアップロード
- 作品編集・削除機能
- 公開/非公開設定
- フィルタリング・検索機能

### 3.3 AI分析機能
- **作品ごとの自動分析**
  - 専門性（強み）分析
  - 文章スタイル・特徴分析
  - 興味・関心領域分析
- **ユーザー総合分析**
  - タグベースの集約・分析
  - 才能スコアの算出
- **分析結果のビジュアライズ**
  - レーダーチャート（専門性）
  - バーチャート（スタイル）
  - タグクラウド（興味関心）

### 3.4 タグベース分析（重点機能）
- AIによる自動タグ提案
- タグの出現頻度分析
- 関連タグのクラスタリング
- タグの時系列変化分析
- インタラクティブなタグマップ

### 3.5 ポートフォリオ表示
- ユーザーダッシュボード
- 作品一覧表示（グリッド/リスト）
- 活動指標の視覚化
- 作品詳細ページ
- 共有可能なURL生成

### 3.6 サブスクリプション管理
- **無料プラン**
  - 基本ポートフォリオ機能
  - 最大10作品までAI分析
- **有料プラン（月額500円）**
  - AI分析の無制限化
  - 詳細AI分析ページ（アナリティクスページ）の開放

## 4. データベース設計

### 4.1 主要テーブル
- **users**: ユーザー情報
- **works**: 作品情報
- **tags**: タグ情報
- **work_tags**: 作品-タグ関連付け
- **ai_analyses**: 作品分析結果
- **user_analyses**: ユーザー総合分析
- **tag_analytics**: タグ分析詳細

### 4.2 リレーションシップ
1. **users ↔ works**: 1対多 (1人のユーザーが複数の作品を持つ)
2. **works ↔ tags**: 多対多 (work_tagsテーブルを介して)
3. **works ↔ ai_analyses**: 1対1 (1つの作品に1つの分析結果)
4. **users ↔ user_analyses**: 1対1 (1人のユーザーに1つの総合分析)
5. **users ↔ tag_analytics**: 1対1 (1人のユーザーに1つのタグ分析)

## 5. 実装状況

### 5.1 タグ分析機能
- **generateTags**: Google Gemini APIを使用して作品からタグを自動生成
- **analyzeTagRelevance**: タグの関連性を分析
- **clusterTags**: 関連タグをクラスタリング
- **analyzeTagTimeline**: タグの時系列変化を分析

### 5.2 API実装
- **generateTagsForWork**: 作品のタグを自動生成するAPIエンドポイント
- **analyzeUserTags**: ユーザーのすべての作品のタグを分析するAPIエンドポイント
- **getWorksByTag**: 特定のタグに関連する作品を取得するAPIエンドポイント
- **getRelatedTags**: 関連タグを取得するAPIエンドポイント
- **getPopularTags**: 人気のタグを取得するAPIエンドポイント

### 5.3 UI実装
- **TagAnalysis**: タグ分析結果を表示するReactコンポーネント
  - タグクラウド表示
  - クラスター表示
  - 関連性チャート
  - タイムライン表示
